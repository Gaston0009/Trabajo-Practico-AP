SNAILRACE API

Bienvenidos a nuestra API nuestros Vehiculos y Marcas

En SNAILRACE , somos una empresa que ofrece datos sobre vehículos (autos y motos) y marcas. Nuestra API te permite acceder a información detallada sobre vehículos, sus precios, características y marcas de todo el mundo. Con nuestra tecnología, podrás realizar búsquedas, filtrados, ordenamientos y mucho más. ¡Todo al alcance de tu mano!

Vehicles API

Aquí puedes interactuar con todos los datos relacionados con vehículos, como autos y motos. A continuación te dejamos las rutas disponibles para acceder a la información:

1. Crear Vehículo

POST /vehicles

Requiere autorización: JWT con rol 'admin'

Descripción: Esta ruta te permite crear un nuevo vehículo en la base de datos. Asegúrate de incluir todos los detalles requeridos.

Ejemplo de solicitud:

                                {
                                    "model": "Toyota Corolla",
                                    "category": "Sedan",
                                    "type": "Car",
                                    "year": 2020,
                                    "fuel": "Gasoline",
                                    "engine": "1.8L",
                                    "price": 20000,
                                    "tags": ["compact", "sedan"],
                                    "brand": "605c72ef153207001f6c72bb"  
                                }

2. Obtener Todos los Vehículos

GET /vehicles

Descripción: Recupera todos los vehículos de la base de datos.

Ejemplo de solicitud:

GET /vehicles

Respuesta: Devuelve una lista de todos los vehículos registrados en el sistema.

3. Obtener Vehículo por ID

GET /vehicles/:id

Descripción: Obtén los detalles de un vehículo específico utilizando su ID.

Ejemplo de solicitud:

GET /vehicles/60b72c33f1a34d2e5c8f3ffb

Respuesta: Información detallada del vehículo con el ID proporcionado.

4. Actualizar Vehículo

PUT /vehicles/:id

Requiere autorización: JWT con rol 'admin'

Descripción: Permite actualizar la información de un vehículo específico.

Ejemplo de solicitud:

                                {
                                    "price": 22000,
                                    "fuel": "Electric"
                                }

Ejemplo de URL: PUT /vehicles/60b72c33f1a34d2e5c8f3ffb

5. Eliminar Vehículo

DELETE /vehicles/:id

Requiere autorización: JWT con rol 'admin'

Descripción: Elimina un vehículo de la base de datos utilizando su ID.

Ejemplo de solicitud:

DELETE /vehicles/60b72c33f1a34d2e5c8f3ffb

Respuesta: Confirmación de la eliminación del vehículo.

6. Filtrar y Ordenar Vehículos por Precio

GET /vehicles/filter/price-sort

Parámetros de consulta:

  • minPrice: Precio mínimo para filtrar.
  • maxPrice: Precio máximo para filtrar.
  • order: Orden de los resultados, puede ser 'asc' (ascendente) o 'desc' (descendente). Por defecto es 'asc'.
  • sortBy: El campo por el cual ordenar, por defecto es 'price'.

Ejemplo de solicitud:

                                GET /vehicles/filter/price-sort?minPrice=10000&maxPrice=30000&order=desc&sortBy=price

Respuesta: Vehículos filtrados y ordenados por precio en el rango solicitado.

Iniciar Sesión y Autenticación

Para acceder a rutas protegidas (como crear, actualizar o eliminar vehículos o marcas), necesitas autenticarte. Esto se hace mediante JWT (JSON Web Token). Primero debes iniciar sesión con un usuario registrado para obtener el token.

1. Iniciar Sesión

POST /auth/login

Descripción: Envía las credenciales del usuario (correo y contraseña) para iniciar sesión.

Ejemplo de solicitud:

                                    {
                                        "email": "admin@snailrace.com",
                                        "password": "admin123"
                                    }

Respuesta exitosa:

                                    {
                                        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
                                        "user": {
                                            "id": "1234567890",
                                            "role": "admin",
                                            "email": "admin@snailrace.com"
                                        }
                                    }

Guarda este token, ya que lo necesitarás para autenticarte en las rutas protegidas.

2. Usar el Token para Autenticarse

Cabecera requerida en rutas protegidas:

                                Authorization: Bearer [tu_token]

Ejemplo con token:

                                GET /vehicles
                                Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Este token verifica tu identidad y permisos. Si eres 'admin', tendrás acceso completo a las rutas protegidas.

Nota: Los tokens tienen un tiempo de expiración. Una vez vencido, deberás volver a iniciar sesión para obtener uno nuevo.

Brands API

A continuación te presentamos las rutas para gestionar las marcas de vehículos:

1. Crear Marca

POST /brands

Requiere autorización: JWT con rol 'admin'

Descripción: Esta ruta permite crear una nueva marca de vehículo.

Ejemplo de solicitud:

                                {
                                    "name": "Toyota",
                                    "country": "Japan"
                                }

2. Obtener Todas las Marcas

GET /brands

Descripción: Obtiene todas las marcas almacenadas en la base de datos.

Ejemplo de solicitud:

GET /brands

Respuesta: Devuelve una lista de todas las marcas registradas.

3. Obtener Marca por ID

GET /brands/:id

Descripción: Obtiene los detalles de una marca específica utilizando su ID.

Ejemplo de solicitud:

GET /brands/60b72c33f1a34d2e5c8f3ffb

Respuesta: Información detallada de la marca con el ID proporcionado.

4. Actualizar Marca

PUT /brands/:id

Requiere autorización: JWT con rol 'admin'

Descripción: Permite actualizar la información de una marca específica.

Ejemplo de solicitud:

                                    {
                                        "country": "USA"
                                    }

Ejemplo de URL: PUT /brands/60b72c33f1a34d2e5c8f3ffb

5. Eliminar Marca

DELETE /brands/:id

Requiere autorización: JWT con rol 'admin'

Descripción: Elimina una marca de la base de datos utilizando su ID.

Ejemplo de solicitud:

DELETE /brands/60b72c33f1a34d2e5c8f3ffb

Respuesta: Confirmación de la eliminación de la marca.

6. Buscar Marca por Nombre

GET /brands/search/name

Parámetros de consulta:

  • name: Nombre de la marca para buscar.

Ejemplo de solicitud:

                                    GET /brands/search/name?name=Toyota

Respuesta: Devuelve la marca correspondiente al nombre proporcionado.
